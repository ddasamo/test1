# Bloom 발문 인지수준 인간 코더 코딩 매뉴얼

본 매뉴얼은 인간 코더가 AI 분석 에이전트와 **동일한 기준**으로 교사 발문을
Bloom 6단계 + 개방성에 따라 분류하도록 안내한다.

대상: 초등 교사 또는 교실담화 분석 경험자
참조 이론:
- Anderson, L. W., & Krathwohl, D. R. (2001). *A taxonomy for learning, teaching, and assessing*.
- Alexander, R. (2008). *Towards dialogic teaching* (5th ed.).

---

## 1. 코딩 절차 (6단계)

1. **전사문 통독** — 코딩 전 전체 한 번 읽고 수업 맥락 파악
2. **분석 대상 추출** — 교사 발화 중 *분류 대상 발문*만 추출 (§3 참조)
3. **Bloom 수준 분류** — 각 발문을 L1~L6 중 하나로 분류 (§4)
4. **개방성 분류** — 각 발문을 open/closed로 별도 분류 (§5)
5. **메타데이터 기록** — 신뢰도/근거/차선 (§6)
6. **검토 후 양식 제출** — 30분 휴식 후 모호 발문 재검토 → CSV 제출 (§7)

⚠️ AI 분류는 코딩 중 참조 금지.

---

## 2. Bloom 분류의 핵심 원칙

**"이 발문이 학생에게 어떤 *사고*를 요구했는가?"**

질문의 *형식*이 아니라 학생이 답하기 위해 *수행해야 할 인지 작업*으로 분류한다.

| 인지 작업 | 수준 |
|---|---|
| 기억에서 사실 끌어내기 | L1 |
| 의미·인과를 이해해서 설명하기 | L2 |
| 절차·공식을 적용하기 | L3 |
| 차이·구조·원인을 분석하기 | L4 |
| 가치·근거를 평가하기 | L5 |
| 새로운 것을 만들어내기 | L6 |

---

## 3. 분석 대상 발문 추출

다음을 **모두** 포함:
- 의문문 또는 응답을 의도한 발화

다음은 **제외**:
- 수사적 질문: "그렇죠?", "맞죠?", "~잖아요?"
- 단순 확인: "이해됐어요?", "들었나요?", "알겠어요?"
- 행동 지시성 질문: "자, 시작할까요?", "준비됐어요?"

⚠️ 제외된 발화는 Bloom 코딩에서 빠지지만, **Flanders 코딩에서는 별도로 처리**된다(주로 ⑤).

## 4. Bloom 6단계 정의 + 한국 초등 교실 단서어

각 수준의 **학술 단서어**와 **초등 교실에서 실제 들리는 표현**을 함께 제시.

### L1 기억 (Remember)
- **인지 작업**: 회상·인식
- **학술 단서어**: 정의해봐, 나열해봐, 식별해봐
- **교실 표현**: 그게 뭐였지?, 기억나?, 이름이 뭐야?, 누구야?, 언제, 어디서, 무엇이
- **예시**:
  - "이 인물의 이름이 뭐였지?"
  - "지난번에 배운 도형의 종류가 뭐였어?"
  - "조선시대 왕의 순서를 말해볼래?"

### L2 이해 (Understand)
- **인지 작업**: 해석·예시화·요약·추론·비교·설명
- **학술 단서어**: 설명해봐, 요약하면, 비교해봐, 추론하면
- **교실 표현**: 이게 무슨 뜻이야?, 왜?, 어떻게 됐어?, 그래서?, 무슨 일이 일어났어?
- **예시**:
  - "왜 이 사건이 일어났을까?"
  - "이 문장이 무슨 의미야?"
  - "그래서 결국 어떻게 됐어?"

### L3 적용 (Apply)
- **인지 작업**: 실행·구현
- **학술 단서어**: 적용해봐, 사용해봐, 구현해봐
- **교실 표현**: 그러면 이걸로 어떻게 할까?, 여기에 써먹으면?, 만약 ~한다면 어떻게 할래?, 이걸 풀어봐
- **예시**:
  - "이 공식을 다른 문제에 어떻게 쓸까?"
  - "배운 방법으로 이 문제를 풀어볼래?"

### L4 분석 (Analyze) ★ 고차 사고
- **인지 작업**: 차이 식별·조직화·귀인
- **학술 단서어**: 분석해봐, 구분해봐, 조직화해봐
- **교실 표현**: 차이가 뭐야?, 공통점은?, 왜 그런 거 같아?, 원인이 뭘까?, 어떤 점이 비슷해?, 근거가 뭐야?
- **예시**:
  - "두 인물의 관점은 어떻게 다른가?"
  - "왜 이런 결과가 나왔을까?"
  - "이 글의 구조는 어떻게 짜여 있어?"

### L5 평가 (Evaluate) ★ 고차 사고
- **인지 작업**: 점검·비판·판단
- **학술 단서어**: 평가해봐, 비판해봐, 판단해봐
- **교실 표현**: 어느 게 더 좋을 것 같아?, 이게 맞을까?, 동의해?, 너라면 어떻게 할래?, 더 나은 게 뭘까?, 왜 그렇게 생각해?
- **예시**:
  - "이 주장의 근거는 충분한가?"
  - "어느 의견이 더 설득력 있어?"
  - "너라면 이 인물의 선택이 옳다고 생각해?"

### L6 창조 (Create) ★ 고차 사고
- **인지 작업**: 생성·계획·산출
- **학술 단서어**: 설계해봐, 산출해봐, 계획해봐
- **교실 표현**: 새로 만들 수 있을까?, 다르게 표현하면?, 다른 방법은 없을까?, 너만의 방법으로 해볼래?, 새로운 ~을 만들어볼래?
- **예시**:
  - "이 문제를 해결할 새로운 방법은?"
  - "다른 결말을 만든다면 어떻게 할래?"
  - "너만의 시를 한 편 써볼래?"

## 5. 개방성 분류 (open/closed)

Bloom 수준과 **별도의 차원**으로 모든 발문에 부여한다.

### 기본 정의
- **닫힌 질문(closed)**: 정답이 한정 (예/아니오, 단일 정답, 회상형)
- **열린 질문(open)**: 다양한 응답 가능, 사고 발산형

### 보조 규칙 (발문 자체로 판별)
| 발문 특성 | 분류 경향 |
|---|---|
| L1 회상 발문 | 본질적으로 closed |
| L2 정답이 있는 이해 발문 | closed |
| L2 다양한 해석 허용 발문 | open |
| L4~L6 발문 | 대부분 open |
| L4~L6 형식이지만 교사가 특정 답을 원하는 경우 | closed |

### ★ 결정적 단서: 교사 후속 발화 패턴
발문만으로 판단이 어려울 때는 **학생 응답을 교사가 어떻게 처리했는가**를 본다.

| 교사의 후속 처리 | 분류 |
|---|---|
| 학생이 다양한 응답을 했을 때 "다른 의견은?", "또?", "왜 그렇게 생각해?", "○○는?"으로 받음 | **open** |
| 학생이 다양한 응답을 했을 때 정답으로 유도 ("다시 생각해봐", "그게 아니라~", "정답은 ~예요") | **closed** |
| 첫 응답이 교사 기대에 맞아 즉시 다음 발문 진행 + 다른 의견 묻지 않음 | **closed** 가능성 ↑ |
| 학생들 의견이 갈려도 모두 수용·확장 | **open** |

근거: Mehan(1979), Cazden(2001) — 발문의 진정한 개방성은 학생 응답을 어떻게 처리했는가에서 드러난다.

## 6. 모호 사례 우선순위 규칙

다음 6가지 규칙을 **순서대로** 적용:

| 순위 | 규칙 | 처리 |
|---|---|---|
| 1 | "왜?"는 다의어 | 직전 맥락 + 학생 응답으로 판단: 회상(L1) / 인과 추론(L2) / 가치 판단(L5) |
| 2 | "어떻게 생각해?" | 의견 요구 → L5, 방법·과정 요구 → L3 |
| 3 | 정답이 1개로 정해진 질문 | 그 정답의 인지수준에 맞춰 분류 |
| 4 | 한 질문이 여러 수준 동시 요구 | **가장 높은 수준**으로 분류 |
| 5 | 학생 응답 보조 검증 | 단답형 → 저차 가능성, 추론·근거·예시 → 고차 가능성<br>(다만 발문 자체가 고차였으면 응답 빈약해도 고차 유지) |
| 6 | 위로 결정 어려움 | `confidence: medium 또는 low` + `alternative` 기록 |

## 7. 신뢰도 + 코딩 양식 (CSV)

### 신뢰도 기준
| 신뢰도 | 기준 |
|---|---|
| **high** | 단서어가 명확하고 인지 작업이 분명 |
| **medium** | 분류는 했으나 차선도 비등하게 가능 |
| **low** | 우선순위 규칙으로 강제 결정. 다른 해석도 합당 |

### CSV 양식 (UTF-8)
```csv
utterance_id,question_text,bloom_level,openness,confidence,rationale,alternative
T042,"두 인물의 관점은 어떻게 다른가","L4","open","high","관점 차이 식별 = 분석",
T067,"왜 그랬을까","L2","open","medium","왜? 다의어, 인과 추론으로 해석","L5"
T028,"이 인물의 이름이 뭐였지","L1","closed","high","단순 이름 회상",
```

### 양식 규칙
- `bloom_level`: `L1`, `L2`, `L3`, `L4`, `L5`, `L6` 중 하나
- `openness`: `open` 또는 `closed`
- `confidence`: `high`, `medium`, `low` 중 하나
- `rationale`: 한 문장 이내, 큰따옴표로 감싸기
- `alternative`: 차선 Bloom 수준 또는 빈칸

## 8. 자주 만나는 경계 사례 FAQ

### Q1. "왜 이렇게 됐을까?"가 L1인지 L2인지?
**A.** *원인을 기억해서 답하는* 발문(역사적 사실 회상 등)이면 L1, *원인을 추론해서 답하는* 발문(미지의 상황 해석)이면 L2. 학생 응답으로 검증.

### Q2. "어느 게 더 좋을까?"가 L3인지 L5인지?
**A.** 정해진 정답이 있는 *방법 선택*이면 L3, *가치 판단 + 근거 제시*를 요구하면 L5. 후속 발문이 "왜?"인지 보고 판단.

### Q3. L4와 L5가 헷갈리는 발문 ("이 글의 주제는 뭐야?")
**A.** *글의 구조에서 주제를 식별*하는 작업이면 L4, *주제의 가치·중요성을 판단*하는 작업이면 L5. 보통 L4.

### Q4. 발문은 L4인데 학생 응답이 단답이면?
**A.** 발문 자체로 분류. 응답 빈약은 학생 요인일 수 있으므로 발문 수준을 낮추지 않는다 (우선순위 규칙 5).

### Q5. 교사가 같은 발문을 반복한 경우?
**A.** 정제 전사문에 별개 발화 번호가 부여돼 있다면 각각 분류. 둘 다 동일 수준으로 코딩.

### Q6. "그래서?", "더 말해줄래?" 같은 짧은 후속 발문(Probing)
**A.** 후속 발문도 분류 대상. 보통 직전 학생 응답의 *수준을 한 단계 끌어올리려는 시도*이므로 L4 이상으로 분류되는 경우가 많다. 다만 단순 "더 말해" 정도면 L2.

### Q7. 학생 자발 질문에 교사가 되묻는 경우 ("그건 너는 어떻게 생각해?")
**A.** 분석 대상. L5 가능성이 높다 (학생에게 평가 요구).

### Q8. Bloom 수준은 L1인데 형식상 열려 있는 질문 ("이 인물에 대해 아는 거 다 말해볼래?")
**A.** L1 + open. 회상이지만 응답 범위가 한정되지 않음. Bloom과 개방성은 별도 차원이다.

## 9. 일치도 산출 통계

본 연구는 다음 지표를 산출한다:
- **Bloom 수준 일치도**: 전체 발문 대상 Cohen's κ
- **개방성 일치도**: open/closed 이진 분류의 일치도
- **저차/고차 이분 일치도**: L1~L3 vs L4~L6 묶음의 κ (해석 단순화)
- **`confidence: high` 한정 부분 일치도**
- **차선 일치도** (1차 또는 차선이 일치하면 일치)

양식 미준수 제출물은 분석 통합에서 제외될 수 있다.

# 방법론적 의사결정 기록 (Methodology Decision Log)

본 문서는 본 연구의 AI 분석 도구(이하 "분석 에이전트") 설계 과정에서
내린 주요 방법론적 의사결정을 기록한다.
**KCI 논문 Methods 섹션에 직접 인용할 수 있도록 학술적 톤으로 작성**한다.

연구 목적
- **RQ1**: 분석 에이전트의 발화 분류 결과가 인간 분석자의 분류와 얼마나 일치하는가
  (inter-rater reliability)
- **RQ2**: 분석 에이전트의 산출물이 교사의 수업 변화 추적에 활용 가능한가
  (longitudinal applicability)

## 결정 1. 분석 이론의 범위 축소 (5개 → 2개)

### 결정
본 연구의 분석 이론을 다음 두 가지로 한정한다.
- **Flanders 언어상호작용분석(FIACS, 1959)**
- **Bloom 신교육목표분류학**(Anderson & Krathwohl, 2001) + 
  Alexander(2008) 대화적 교수법의 개방성·연속성 분석

### 근거
1. **인간 코더 부담의 현실성**: 본 연구의 인간-AI 일치도 검증은
   최소 2인 이상의 독립 코더가 동일 전사문을 분류해야 한다.
   5개 이론(범주 약 30~40개)을 동시 코딩하는 것은
   현실적 시간·인지 부담을 초과한다.
2. **이론 간 변량 분리**: Flanders(거시적 발화 구조 + 영향력 방향)와
   Bloom(미시적 인지 수준)은 분석 대상 차원이 명확히 구분되어
   중복 변량이 최소화된다.
3. **한국 교실연구 표준성**: 두 이론 모두 한국 교실연구에서
   수십 년간 적용된 코딩 매뉴얼이 정착되어 있어,
   초등 교사 출신 인간 코더 모집과 신뢰도 확보에 유리하다.
4. **AI 난이도 차이의 방법론적 가치**: Flanders는 비교적 구조적·표층적 분류,
   Bloom은 의미·맥락 의존적 분류로 AI에게 요구되는 추론 깊이가 다르다.
   두 이론에서의 일치도 차이를 비교하는 것 자체가 방법론적 기여가 된다.

### 향후 확장 보존
나머지 3개 이론(IRF, Wait Time, MET 피드백)의 프롬프트와 코드 슬롯은
저장소에 보존하되 본 논문 분석에서는 사용하지 않는다.

## 결정 2. 점수화·등급화의 전면 제거

### 결정
모든 이론 프롬프트에서 다음을 제거하였다.
- 채점 기준표 (예: "고차 사고 30%↑ = 4점")
- 점수 합산 출력 (예: "나의 점수: _점")
- 등급 라벨 (예: "우수", "양호", "제한적")

### 근거
1. **이론적 충실성**: Flanders(1959), Bloom(2001) 원전(原典)은
   분류·집계·해석을 제시할 뿐 점수화 체계를 포함하지 않는다.
   임의의 점수 구간은 분석자 가공이며 이론에서 도출된 임계점이 아니다.
2. **일치도 측정의 타당성**: 일치도 산출은 *분류 결과의 일치*에 기반한다.
   점수 자체는 측정 대상이 아니며, 점수 산출은 분류 결과의 후처리에
   불과해 일치도 검증에 불필요한 의존성을 만든다.
3. **연구자 처방과 객관 정보의 분리**: 점수는 "이 수업이 우수한가"라는
   판단을 분석 도구가 내포하게 만들어, 분석의 객관성을 훼손한다.
   본 연구는 분류 결과(객관)와 처방 제안(주관)을 명시적으로 분리한다
   (결정 4 참조).

### 보존 항목
다음은 *객관 정보*로 보고하며 점수화하지 않는다.
- 분포표(개수, 비율)
- i/d 비율, 수용·격려 비율, 고차 사고 비율 등 이론 핵심 지표
- 시각화(ASCII 막대그래프, Markdown 표)

## 결정 3. 분류 결과의 메타데이터 강화 (신뢰도·근거·차선)

### 결정
모든 발화·발문 분류 결과에 다음 세 메타데이터를 부착한다.
- **confidence**: `high` / `medium` / `low` 3단계
- **rationale**: 분류 근거를 짧은 문자열로 명시
- **alternative**: 차선(second-best) 범주, 없으면 `null`

### 근거
1. **분류 모호성의 명시적 표현**: Flanders 10범주와 Bloom 6단계 모두
   경계 사례(예: "왜?"가 회상인지 추론인지)가 존재한다.
   기존 연구는 이를 강제 단일 분류로 처리했으나, 본 연구는
   분류자의 확신도를 메타데이터로 기록한다.
2. **일치도 분석의 해상도 향상**: 일치도(Cohen's κ 등)를 전체 발화 대상으로
   계산할 뿐 아니라, `confidence: "high"` 발화에 한정한 부분 일치도도
   별도로 산출하여 *어느 영역에서 AI-인간 불일치가 발생하는가*를
   세분화할 수 있다.
3. **분류 근거의 추적 가능성**: `rationale` 필드는 AI 분류의 *설명 가능성*을
   확보하여, 인간 코더가 동일 발화를 재검토할 때 분류 기준을 비교할 수 있다.
4. **차선 범주의 활용**: `alternative` 필드를 통해 *부분 일치* 분석이 가능하다.
   AI의 1차 분류가 인간 코더와 다르더라도 차선 범주가 일치한다면
   "근사 일치(approximate agreement)"로 분류할 수 있어, 이론의 미세한
   경계 사례에서 발생하는 불일치의 본질을 진단할 수 있다.

### 신뢰도 3단계의 정당화
2단계(high/low)는 정보량이 부족하고, 5단계는 분류자 부담이 과도하다.
3단계는 분류자가 직관적으로 구분 가능하며, 통계적 분포 분석에도 적합하다.

## 결정 4. 객관 분석과 처방 제안의 분리

### 결정
이론별 출력 JSON 스키마에서 처방적 코멘트는 별도 필드로 격리한다.
- **객관 분석**: `classifications`, `stats` (RQ1 일치도 검증의 분석 단위)
- **처방 제안**: `diagnosis.intervention_suggestion` (RQ2 수업변화 추적의 분석 단위)

원래 프롬프트의 "다음 수업에서 시도할 1가지" 항목은 명칭을
`next_action`에서 **`intervention_suggestion`**으로 변경하여
처방적 성격을 명시한다.

### 근거
1. **RQ1과 RQ2의 분리**:
   - RQ1(일치도) 검증 시에는 *분류 결과만* 사용 →
     처방 코멘트는 일치도 산출에서 제외
   - RQ2(수업변화) 추적 시에는 *지표 변화 + 처방 코멘트* 모두 분석 가능
2. **분석 도구의 책무 한계 명시**: 처방 코멘트는 분석 도구의 *추론적 제안*이며,
   객관 분류와 동등한 신뢰성을 갖지 않는다. 이를 별도 필드로 분리함으로써
   사용자(현장 교사) 및 후속 연구자가 두 유형의 출력을 구분 활용할 수 있다.
3. **현장 가치와 학술적 엄밀성의 양립**: 처방 코멘트는 현장 교사에게
   실질적 가치를 제공하므로 제거하지 않는다. 다만 학술 분석에서는
   별도 처리한다.

## 결정 5. 한국어 단서어 매뉴얼 명시 (Bloom)

### 결정
Bloom 6단계 분류 프롬프트에 영어권 원본의 단서 동사(verb)를
한국 초등 교실 발화 특성에 맞게 보강한다.
- 영어 학술 단서어 + 한국어 일상·교실 표현 병기
- 예: L1 "정의해" + "그게 뭐였지?", "기억나?"
- 예: L5 "평가해" + "어느 게 더 좋을 것 같아?", "이게 맞을까?"

### 근거
1. **AI-인간 동일 매뉴얼 사용**: 인간 코더와 AI가 *동일한 한국어 단서어 매뉴얼*을
   참조함으로써, 일치도 측정의 외적 타당도(external validity)를 확보한다.
2. **초등 발화 특성 반영**: 초등 교사 발문은 학술적 동사보다 일상 한국어 표현이
   다수이므로(예: "분석해봐" → "왜 그런 거 같아?"), 원전의 영어 동사 목록만으로는
   분류 누락이 발생한다.
3. **재현 가능성**: 본 매뉴얼은 프롬프트 파일(`prompts/bloom.md`)에 명시적으로
   기록되어, 후속 연구가 동일 매뉴얼로 재현·검증할 수 있다.

## 결정 6. 개방성 분류의 외부 증거 활용 (Bloom)

### 결정
발문의 개방성(open/closed) 분류 시 발문 자체뿐 아니라
**교사 후속 발화 패턴을 보조 증거**로 활용한다.
- 학생이 다양한 응답을 했을 때 교사가 "다른 의견은?", "또?", "왜?"로 받음 → **open**
- 학생이 다양한 응답을 했을 때 교사가 정답으로 유도("다시 생각해봐", "그게 아니라~") → **closed**

### 근거
1. **개방성의 조작적 정의 문제**: 발문 자체만으로는 "정답 유도 여부"를 판별하기
   어렵다. 형식상 열린 질문도 교사가 단일 정답을 기대하는 경우가 다수다.
2. **교사 발화의 *후속 처리*가 결정적**: Mehan(1979), Cazden(2001) 등이
   지적한 바와 같이, 발문의 진정한 개방성은 *학생 응답을 어떻게 처리했는가*에서
   드러난다. 본 연구는 이 관찰을 분류 규칙에 반영한다.
3. **자의성 감소**: 후속 발화라는 외부 증거를 도입함으로써 분류자의 자의성을
   줄이고, AI-인간 일치도에서 가장 흔히 발생하는 개방성 불일치를 완화한다.

## 결정 7. STT + 화자 분할 파이프라인 선택

### 결정
- **STT**: faster-whisper (large-v3 모델)
- **화자 분할**: pyannote.audio 3.x (`pyannote/speaker-diarization-3.1`)
- **결합**: 단어 수준 타임스탬프(Whisper) + 화자 구간(pyannote)을
  시간 기준으로 정렬

### 근거
1. **객관적 화자 식별의 우선성**: LLM 기반 텍스트 휴리스틱(발화량/단서어)
   만으로 화자를 추정하면, 학생이 길게 발표한 구간에서 오분류가 발생한다.
   음향 기반 화자 분할은 음성학적 특징을 사용하므로 텍스트 휴리스틱보다
   화자 식별의 객관성·재현성이 높다.
2. **한국어 정확도**: Whisper large-v3는 한국어 STT에서 오픈소스 모델 중
   가장 높은 정확도를 보고하고 있다(OpenAI, 2023).
3. **연구 재현성**: 두 도구 모두 오픈소스이고 모델 버전 고정이 가능하여
   후속 연구가 동일 파이프라인을 재현할 수 있다.

### 화자 라벨링 후처리 규칙
pyannote의 SPEAKER_00, SPEAKER_01… 라벨을 T(교사) / S1, S2, … (학생) /
SS(다수 동시) / (침묵 N초)로 변환할 때 다음 규칙을 적용한다.
1. 전체 발화 시간 점유율 최다 화자 = 교사 후보 1순위
2. 후보가 교사 단서어("자,", "친구들", 학생 호명, 전환 신호) 빈도 1위 →
   교사(T)로 확정
3. 1·2가 불일치하면 단서어 우선 (학생이 발표·연설로 시간 점유 상위인 경우 대비)
4. 발화 간 간격 ≥ 3.0초 → `(침묵 N초)` 라인 자동 삽입 (Rowe 기준)
5. 두 화자 이상 음성이 겹치는 구간 → SS

## 결정 8. 분석 엔진(LLM) 선택

### 결정
- **분석 LLM**: Claude Opus 4.7 (`claude-opus-4-7`)
- 호출 시 prompt caching으로 system 프롬프트 캐싱

### 근거
1. **한국어 이해와 분류 일관성**: 본 연구의 분류 정확도는 한국어 의미 이해와
   장문 일관성에 결정적으로 의존한다. Claude Opus 4.7은 두 측면에서
   현존 최고 수준의 성능을 보고하고 있다.
2. **재현성 확보**: 모델 ID와 버전을 고정(`claude-opus-4-7`)함으로써
   동일 입력에 대한 결과 재현이 가능하다.
3. **장문 컨텍스트**: 20~40분 수업 전사(약 3,000~6,000 토큰)와 이론별
   프롬프트(약 2,000~3,000 토큰)를 함께 처리하기에 충분한 컨텍스트 윈도우.

### 분석 결과의 안정성 확보
같은 전사문에 대해 LLM이 호출마다 분류가 미세하게 달라질 수 있으므로,
본 연구는 다음 절차를 적용한다.
- 동일 전사문에 대해 LLM 분석을 **3회 반복**
- 각 발화·발문별로 3회 중 다수결(majority vote)을 최종 AI 분류로 채택
- 3회 모두 다른 경우(=낮은 일관성)는 `confidence: "low"`로 표시
- 이 절차는 LLM 자체의 *test-retest reliability*를 평가하는 데이터로도 활용

## 분류·산출 흐름 (요약)

```
[교사 mp3 업로드]
       │
       ▼
[faster-whisper large-v3]  ─ 단어 수준 전사 + 타임스탬프
       │
       ▼
[pyannote.audio 3.1]       ─ 화자 구간 분할
       │
       ▼
[refine.py]                 ─ T/S 라벨링, SS, 침묵 삽입
       │
       ▼
[정제 전사문]
       │
       ├─→ [Flanders 프롬프트 × Claude Opus 4.7 × 3회] ─→ majority vote ─→ JSON
       └─→ [Bloom 프롬프트 × Claude Opus 4.7 × 3회]    ─→ majority vote ─→ JSON
              │
              ▼
       [인간 코더 분류 결과와 Cohen's κ, Krippendorff α 계산]
       [부분 일치도 (high-confidence 한정) 별도 산출]
       [차선 범주 활용 근사 일치도 별도 산출]
```
